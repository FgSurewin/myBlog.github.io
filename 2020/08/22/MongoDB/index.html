查询数据

在 MongoDB 中, 使用对象作为where关键字能够更加快速, 更直观的去获取数据.

MongoDB 的查询方式以链式的方式呈现. 在查询中, 我们可以使用sort, limit等关键字进行对数据的操作.

在 MongoDB 中使用 sort() 方法对数据进行排序，sort() 方法可以通过参数指定排序的字段，并使用 1 和 -1 来指定排序的方式:

  • 1 为升序排列
  • -1 为降序排列。

CRUD

基本的 API 与原生 MongoDB 几乎一致, 注意以下几点即可:

  • API 都是 Promise, 可以选择使用回调函数或者使用await
  • update不需要使用$set关键字也能进行更新

能使用 params 是因为路由使用了动态路由

补充知识

补充知识部分是对RouteBuilder进行个别知识点的补充(源码地址). 在编写简易 CRUD 的模板的时候, 采用传统MVC模型的文件结构方法. 即:

  • controller负责整体调节的功能
  • service负责调动其model
  • model则是负责关联 Databse

在这种目录格式下管理代码, 优点是逻辑清晰, 可维护性高. 缺点是很多臃肿, 重复的代码. 从传统 RestFul API 的角度来看, routes文件完全可以自行生成. 我们只需要关注对应的controller即可.

整个设计思路是通过controller的配合来实现的. 利用其对象声明其关系.

this 的理解

  • this 的默认指向是 window(Node 环境下是 global)
  • 普通内嵌函数会丢失 this(因为词性环境已经不同)

值得注意的是for循环并不是函数
