What is linked list

Posted on 2020-08-05 In programming , data-structure Valine:

前言

链表是一种很有趣的数据结构, 能够动态的存储地址和内容. 链表的原理和我们平时的闯关游戏是一样的, 我们只有通过当前关卡才能知道下一关是什么.

当我们谈论链表时, 我们经常会将其与数组进行比较. 根据链表的特性, 我们不难猜出, 链表的优势在于增与删. 因为都是通过指针去指向下一个节点, 只要我们改变指向的位置就能轻松完成任务. 并不如数组一样麻烦, 需要挪动位置(假设插入的位置不是首尾).

事实上, 在 JavaScript 中操作数组并没有想象中那么麻烦. 我们可以使用各种内置 API 来完成我们的设想. 实际上 JavaScript 中的数组并不是真正意义上的数组. 它也是一个对象. 可以理解成披着数组皮的对象. 我们都知道 JS 存在原型链, 而我们所谓的 Class 或者 Constructor 就是我们对于原型的利用. 一切都有原型, 最终的原型便是 Object 本身. 所以我们可以说 JavaScript 的一切都是源于对象. (当然, 我们的基本数据类型不算).

链表的形式有很多种: 单向链表, 双向链表和环形链表. 其他类型的链表都是由单向链表延伸的.

从图中我们可以看出链表的组成分为两个部分:

  • 存放当前节点内容的空间
  • 存放下一个节点的地址的空间

我们可以用一个简单的 class 来创建每一个节点

其实, 对于数据的使用不过是对于的数据的CRUD. 不同的数据结构有不同的优缺点.

对于插入数据, 我们可以选择在右边插入新的数据, 这时候head就永远是头部. 也可以选择在左边插入新的数据, 这时候head就是永远是尾部.

我们只要将this.head作为next的参数不断传递到该函数即可.

当我们需要往右边的插入新的节点时, 我们需要寻找到最后的节点, 然后将这个最后节点的next指向新的节点的地址. 要寻找最后的节点, 我们有两种方法:

  • 递归
  • 循环

循环的难点在于如果我们需要特定的 index 的时候, 如何设置结束点. 应该取<还是<=. 而递归的难点在于如何设定ending point

在特定位置插入新的节点就是改变前者的指针的位置指向新的节点, 然后将新的节点的指针指向位置指向后者. 这也是链表为什么在增与删的方面比数组优势的地方. 因为完全不需要其他多余步骤, 只关注指针指向位置.

更新节点的方法没什么难度.

删除节点的方法在于要分清界限, 不同的范围做不同的事.

环形链表的拓展

正如我前文提及, 链表还有环形链表的结构. 其实就是将最后的节点指向链表中的某个节点. 如果指向头部节点, 则是全循环(一个圆圈).

如何判断是否有环

利用 Hash Table

可以利用 Hash Table 的方式来记录每一个已经走过的节点, 当节点开始重复的时候, 我们就知道存在环形.

利用双指针

上面的方法的缺点就是空间复杂度可能会比较大, 所以我们更加取巧的使用快慢指针来查是否存在环形.

假设两个球(绿色球和黑色球)同时从蓝点出发 :

  • 黑球球的速度为每次一步
  • 绿球的速度为每次两步

通过模拟你会发现, 最后两球会在箭头指向的方向相遇. 当两球能够相遇, 则证明该链表存在环形.
