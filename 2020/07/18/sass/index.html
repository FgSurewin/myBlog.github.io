进阶

此进阶篇仅对于笔者而言. 因为我一直以为简单的使用变量, 嵌套便算是学习了 Sass, 这种想法太愚蠢了. Sass 的强大超出我的认知.

虽然是讨论 Sass, 但是基本的 CSS 的知识还是补充学些的. 在讨论&字符的时候, 我们需要明白 CSS 选择器的一些特点:

@mixin的优势有两个:

  • 可以传递整个 CSS 样式, 包括其他样式名字
  • 可以传递参数, 可以传很多个参数

举一个实战项目的例子, 假设我们需要完成一个全局样式, 点击切换的效果, 我们有多种方式可以选择, 其中我们可以选择预设样式(即我们预先写好几套样式供给用户选择). 预设样式虽然麻烦, 但是这样的 CSS 效果会更加贴合 UI 的设计. 缺点是文件的容量会变大, 因为有多套样式.

整个案例执行起来代码量其实没有我们想象中那么负责, 虽然预设样式听起来很吓人. 我们会利用 CSS 选择器中, 属性选择器作为突破口. 配合 Sass 的强大的语法功能来完成.

接下来我们将会学习 Sass 的更多功能来完成整个案例.

整个案例得难点在于如何理解@mixin

因为是 CSS 中的父与子的关系, 所以 HTML 部分只需要在最外层添加该样式即可. 这里我们通过绑定在 Vuex, 这样的话其他组件也能轻易改变整个页面的样式

看起来样式其实很麻烦, 其实并不麻烦. 因为样式的改变主要关注在字体颜色, 背景颜色, 边框颜色三个关于颜色的部分. 而我们在写 theme 的 map 的时候, 其实所有的 key 值是一样的, 就是说我们调用@include传递的参数是一样的, 取决于我们的基础样式的配置结果. 在本例子中, 我只配置了一个bg-primary, 但是实际上可以配饰很多, 比如active-color之类的.

所以我们只需要根据 UI 去配置多套样式, 只需在_theme文件处理后, 后面无论样式的设计,还是逻辑都和我们平时的写法一摸一样. 只需要更换data-theme=val即可.
